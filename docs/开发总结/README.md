# 开发总结文件夹

## 用途说明
此文件夹存储AI开发过程中的时间戳总结文件，用于：
- 记录每次开发的具体内容和变更
- 追溯问题和解决方案
- 积累开发经验和教训
- 为开发者提供项目演进历史

## 文件命名规范
`YYYY-MM-DD_HH-mm_[具体内容描述].md`

示例：
- `2025-01-15_14-30_Epic1-Story1完成.md`
- `2025-01-15_16-45_Epic1-Story2进度50%.md`
- `2025-01-16_10-15_Epic1完成总结.md`

## 重要说明
- 这些文件是**备份和追溯用途**，不参与开发流程
- AI每次完成工作后必须创建对应的总结文件
- 开发者可查看这些文件了解项目开发历史
- 遵循标准格式确保信息一致性
